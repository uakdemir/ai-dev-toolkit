**Editing native code (`ios/` or `android/`).**

This project uses the Expo managed workflow. The `ios/` and `android/`
folders are typically generated by `npx expo prebuild` and should NOT be
hand-edited unless you have deliberately left the managed workflow.

Checklist before editing:
1. Has `npx expo prebuild` already been run for this project? If not,
   STOP — native edits will be lost on the next prebuild.
2. Is there a managed-workflow alternative (Expo config plugin,
   `app.json` entry)? Prefer it.
3. If a native edit is truly required, document it in
   `CLAUDE.local.md` so the next developer knows why.
